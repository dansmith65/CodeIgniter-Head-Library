CodeIgniter Head Library

Head Library v2.1

The head library has one purpose: to create a valid head area for an HTML document. It renders everything from the doctype to the closing head tag, and allows you to do it with as little clutter as possible from your controllers.

You can download this Head Library at its GitHub page. A sample config file is also included.

The original Head Library by Adam Fairholm is available here.

Usage

The head library should be auto-loaded. You can specify defaults to be loaded when the library is, and change defaults/add resources at run-time. Refer to Configuration to see the the available defaults. Refer to Function Reference to see what resources can be added at runtime.

Output

The head library can be used into two ways. By default, a constant called HEAD is defined that can be called in a view or controller. The name of the defined constant can be changed using a config variable called constant_name.

echo HEAD;

However, the library can be set to have the render_head() function return a string.

$data['head'] = $this->head->render_head();

This config item is called output_string.

Configuration

The head library has a range of available configuration options. Class variables are set to default values, and can be overridden by a head array config in the config/head.php file.

The initialize() function can be called with a config array.

$config['title'] = "Test Page";
$config['debug'] = TRUE;

$this->head->initialize($config);

The main render_head() function also accepts a config array.

$config['title'] = "Test Page";
$config['debug'] = TRUE;

$this->head->render_head($config);

Values can also be set individually.

$this->head->title = "Test Page";

Below is a table of available configuration variables.

Debugging

Important:  debugging does not work with the current version of Head library

Many times, its useful to be able to quickly check if all the resources you are loading actually exist. For example, you could be trying to figure out why a Javascript function isn't working, and rather than looking into the head and figuring out if any of the links are wrong, you can just tell the head library to check for you. If you don't have something like an in-browser error console or another similar tool, this could come in handy.

Debugging is set to FALSE by default, but set it to TRUE to turn it on. If there are problems, it will let you know in the form of a CodeIgniter error message.

The head debugger checks for:

  • CSS Files
  • JS Files
  • Favicon Files
$this->head->debug = TRUE; The following errors were encountered in the head area:

master.css not found
coda-slider.css not found

Important:  The debug feature checks for the actual file, so if you are loading resources that aren't files (like a CSS function that is routed through the CI architecture), then this won't help you.

Doctypes

Doctypes are created using the doctype() function from the Code Igniter native html helper. If a doctype is selected that is not supported, an error message will be generated if errors are allowed.

Base URLs

Although not necessary for some doctypes, the head library has the ability to generate a base tag from the configuration values listed above. More information on the base tag can be found here.

A great overview of how the base tag is useful for CI organization can be found here.

Important:  This will not affect relative links used for CSS, JS, and favicon. See File References for more info.

Meta Data

By default, five meta data options are handled individually and can have default values set through a config file or set individually. These options are:

  • Content type
  • Content language
  • Author
  • Description
  • Keywords

Additional meta tags can be added with the add_meta() function.

Google Analytics

Google Analytics now requires tracking code to be placed before closing head tag, not the closing body tag as in past versions. You can automatically add this code in the correct place by providing your Google Analytics tracking ID via the variable ga_tracking_id.

Packages

Often times, pages require a number of CSS and JS files that need to be loaded in order for a certain function to work correctly. For example, the jQuery Coda Slider requires five Javascript files and a CSS file. The head library offers an easy way to accomplish this by creating named "packages" that specify what files should be loaded. These packages reduce the amount of code needed to determine what files need to be called.

Packages are defined in the config file by adding on to a variable called $config['packages']. Here is an example of a package for the jQuery Coda Slider.

$config['packages']['coda_slider']['css'][] = 'coda-slider.css';
$config['packages']['coda_slider']['js'] = array(
    '$config['jquery']',
    'jquery.scrollTo.js',
    'jquery.localscroll.js',
    'jquery.serialScroll.js',
    'coda-slider.js'
);

Packages can contain references to other packages. Each referenced package will be recursively loaded when the main package is loaded. Below, 'jquery' has been defined as a package, and referenced by the 'coda_slider' package:

$config['packages']['jquery']['js'][] = $config['jquery'];

$config['packages']['coda_slider']['packages'][] = 'jquery';
$config['packages']['coda_slider']['css'][] = 'coda-slider.css';
$config['packages']['coda_slider']['js'] = array(
    'jquery.scrollTo.js',
    'jquery.localscroll.js',
    'jquery.serialScroll.js',
    'coda-slider.js'
);

Packages can contain any of these elements: packages, meta, css, js, jquery, inline_css, inline_js, feed, misc.

$config['packages']['test']['packages'][] = 'coda-slider';
$config['packages']['test']['meta'][] = array('robots', 'no-cache');
$config['packages']['test']['css'][] = array('/styles.css', 'screen');
$config['packages']['test']['js'][] = 'java.js';
$config['packages']['test']['jquery'][] = '$(#slide_down_div).slideDown("slow");';
$config['packages']['test']['inline_css'][] = '* {font-family: Arial;}';
$config['packages']['test']['inline_js'][] = 'BASE_URL = "'.base_url().'";';
$config['packages']['test']['feed'][] = array('http://www.example.com/news.xml', 'Site News');
$config['packages']['test']['misc'][] = '<!-- miscellaneous stuff -->';

The format from prior version of Head library still works, but only allows for css and js files:

$config['packages']['coda_slider'] = array(
    'coda-slider.css',
    'jquery.js',
    'jquery.scrollTo.js',
    'jquery.localscroll.js',
    'jquery.serialScroll.js',
    'coda-slider.js'
);

Now that the package is defined, all you have to do is load it. You can do this by adding it through the add_package() function or you can Load Packages by Default.

Defaults

You can load any item by default by adding it to the defaults array in the config file.

$config['defaults']['packages'][] = 'coda-slider';
$config['defaults']['meta'][] = array('robots', 'no-cache');
$config['defaults']['css'][] = array('/styles.css', 'screen');
$config['defaults']['js'][] = 'java.js';
$config['defaults']['jquery'][] = '$(#slide_down_div).slideDown("slow");';
$config['defaults']['inline_css'][] = '* {font-family: Arial;}';
$config['defaults']['inline_js'][] = 'BASE_URL = "'.base_url().'";';
$config['defaults']['feed'][] = array('http://www.example.com/news.xml', 'Site News');
$config['defaults']['misc'][] = '<!-- miscellaneous stuff -->';

Loading Packages by Default

You can load packages by default by adding a packages array node to the defaults array.

$config['defaults']['packages'] = array('coda_slider', 'another_package');

Function Reference

$this->head->initialize([config])

Any configuration option can be overwritten by calling this function with an array of config options.

Also refer to render_head() for another method of overwritting default configuration.

$config['title'] = 'Test Page';
$config['debug'] = TRUE;

$this->head->initialize($config);

$this->head->render_head([config])

This function creates the head content from the configuration in the config file, the configuration passed to it, and the values set via the various other functions. MUST be called for the head content to be created.

Also refer to initialize() for another method of overwritting default configuration.

$this->head->render_head() $config['title'] = 'Test Page';
$config['debug'] = TRUE;

$this->head->render_head($config) // when option output_string is TRUE

$data['head'] = $this->head->render_head();

$this->head->add_package(name)

Add package to head that is defined in configuration file.

$this->head->add_package('jquery_ui'); $packages[] = 'jquery_ui';
$packages[] = 'jquery_validate';

$this->head->add_package($packages) $this->head->add_package('jquery_ui|jquery_validate')

$this->head->include_packages()

Alias of add_package()

$this->head->add_meta(name, value, [equiv or name='name'])

Uses the meta function from the Code Igniter native html class.

$this->head->add_meta('robots', 'no-cache');

$this->head->add_meta('expires', 'Mon, 22 Jul 2002 11:12:01 GMT', 'equiv');

Produces:

<meta name="robots" content="no-cache" />

<meta http-equiv="expires" content="Mon, 22 Jul 2002 11:12:01 GMT" />

$this->head->add_css(file, [media = 'all'], [condition])

Add link to css file. A search for "html conditional comments" will explain what the condition parameter is used for.

File References gives detailed information on how the file parameter is handled.

$this->head->add_css('styles.css');

$this->head->add_css('styles.css', 'screen');

$this->head->add_css('styles.css', 'all', 'IE');

Produces:

<link type="text/css" href="http://example.com/ci/assets/css/styles.css" rel="stylesheet" media="all" />

<link type="text/css" href="http://example.com/ci/assets/css/styles.css" rel="stylesheet" media="screen" />

<!--[if IE]>
<link type="text/css" href="http://example.com/ci/assets/css/styles.css" rel="stylesheet" media="all" />
<![endif]-->

Examples assume codeigniter is installed in the root as 'ci' and css_location option is left as default 'css/'.

$this->head->add_js(file, [condition])

Add link to javascript file. A search for "html conditional comments" will explain what the condition parameter is used for.

File References gives detailed information on how the file parameter is handled.

$this->head->add_js('java.js');

$this->head->add_js('java.js', 'lt IE 7');

Produces:

<script type="text/javascript" src="http://example.com/ci/assets/js/java.js"></script>

<!--[if lt IE 7]>
<script type="text/javascript" src="http://example.com/ci/assets/js/java.js"></script>
<![endif]-->

$this->head->add()

Can be used instead of add_css() or add_js(). The type will be determined based on the file extension (.css = styles, .js = javascript, all other file extensions will produce an error).

$this->head->add('styles.css', 'all', 'IE');
$this->head->add('java.js', 'lt IE 7');

$this->head->add_jquery(code)

Add a line or block of jQuery code that is rendered in the head with the $(document).ready function. jQuery file will be loaded automatically when this function is used.

(Dont forget to name your jQuery file using the jquery_file config variable!)

$this->head->add_jquery('$(#slide_down_div).slideDown("slow");');

Produces:

<script type="text/javascript" src="http://example.com/ci/assets/js/jquery.js"></script>
<script type="text/javascript" language="javascript">
// <![CDATA[
$(document).ready(function(){
    $(#slide_down_div).slideDown('slow');
});
// ]]>
</script>

Examples assume codeigniter is installed in the root as 'ci', js_location option is left as default 'js/', and jquery_file option is left as default 'jquery.js'.

$this->head->add_inline_css(code)

Add inline CSS to head.

$this->head->add_inline_css('* {font-family: Arial;}');

Produces:

<style type="text/css">
    * {font-family: Arial;}
</style>

$this->head->add_inline_js(code)

Add inline JavaScript to head.

$this->head->add_inline_js('BASE_URL = "'.base_url().'";');

Produces:

<script type="text/javascript" language="javascript">
// <![CDATA[
    BASE_URL = "https://example.com/ci/";
// ]]>
</script>

$this->head->add_inline(code, css or js)

Performs same function as add_inline_css() and add_inline_js(). Requires either 'css' or 'js' as the second parameter.

$this->head->add_inline('* {font-family: Arial;}', 'css'); $this->head->add_inline('BASE_URL = "'.base_url().'";', 'js');

$this->head->add_feed(URL, name, [rss or atom = 'rss'])

Add RSS or Atom feeds to head. The optional third parameter determines if the feed is listed as RSS or Atom with RSS as the default value.

$this->head->add_feed('http://www.example.com/news.xml', 'Site News');
$this->head->add_feed('http://www.example.com/events.xml', 'Site Events', 'atom');

Produces:

<link href="http://www.example.com/news.xml" type="application/rss+xml" rel="alternate" title="Site News" />
<link href="http://www.example.com/events.xml" type="application/atom+xml" rel="alternate" title="Site Events" />

Important:  The URLs for the add_feed() function should be full URLs.

$this->head->add_misc(code)

Add anything you want to the head. code will be added exactly as provided.

$this->head->add_misc('<style type="text/css">body {background-color:yellow}</style>');

Produces:

<style type="text/css">body {background-color:yellow}</style>

File References

Important:  This section only applies to linked CSS, JS, and favicon.

If a file reference is a full URL or begins with a forward slash, then it is not changed:

$this->head->add_css('http://example.com/assets/css/styles.css');

$this->head->add_css('/styles.css');

Produces:

<link type="text/css" href="http://example.com/assets/css/styles.css" rel="stylesheet" media="all" />

<link type="text/css" href="/styles.css" rel="stylesheet" media="all" />

Otherwise...

CSS or JS file references are prepended with config values: 'base_url' and 'css_location' or 'js_location'.
(Examples assume codeigniter is installed in the root as 'ci' and css_location/js_location options are left as default)

$this->head->add_css('styles.css');

$this->head->add_js('java.js');

Produces:

<link type="text/css" href="http://example.com/ci/assets/css/styles.css" rel="stylesheet" media="all" />

<script type="text/javascript" src="http://example.com/ci/assets/js/java.js"></script>

favicon_location will be prepended with config value: 'base_url'.
(Examples assume codeigniter is installed in the root as 'ci')

$config['use_favicon'] = TRUE;
$config['favicon_location'] = 'assets/images/favicon.ico';

Produces:

<link rel="shortcut icon" href="http://example.com/ci/assets/images/favicon.ico" />

Therefore, the <base> tag will have no effect on links to CSS, JS, or favicon.
